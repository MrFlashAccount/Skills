# Contrastive Examples

Use this file when a task needs quick good-vs-bad framing rather than abstract principles alone.

## Example: marketing first screen

Stronger:
- one clear claim
- one visible proof vector
- composition with a dominant read path
- enough air for the promise to feel confident

Weaker:
- headline, subheadline, badges, feature row, testimonial, and screenshot all competing at once
- heavy effects used to simulate importance

## Example: dashboard grouping

Stronger:
- controls, data, and actions grouped by job
- density tuned for repeat use
- cards only where containment improves comprehension
- container geometry feels intentional and supports the surrounding tone

Weaker:
- marketing spacing on a utility surface
- every metric inside another decorative box
- primary controls visually tied with secondary filters
- square or commodity-default containers that flatten the rest of the visual language

## Example: docs surface

Stronger:
- prose, code, warnings, and navigation clearly separated by role
- reading rhythm stays calm across long pages

Weaker:
- docs page styled like a launch page
- low contrast between prose and code responsibilities

## Example: navigation

Stronger:
- primary and secondary actions are grouped clearly
- navigation placement supports the page composition
- active location and next moves are easy to read quickly

Weaker:
- nav chrome is more expressive than the information structure
- menu reveal behavior becomes the main event
- everything is packed into one flat strip with weak grouping

## Example: typography

Stronger:
- hierarchy comes from the full text system, not one giant line
- expressive display treatment stays readable and supports the content model
- type choices match the product class and screen density

Weaker:
- the hero heading does all the work while the rest collapses
- text effects try to fake distinctiveness
- display choices look impressive in a screenshot but weaken real reading

## Example: media

Stronger:
- one dominant asset sets the section's payoff, with secondary media clearly subordinate
- screenshots and product shots are framed to support the surrounding claim
- the first thing to look at is obvious before any carousel or gallery interaction

Weaker:
- five equally loud assets compete with no clear lead
- media is dropped in because the section felt too empty
- carousel or gallery mechanics hide the one proof point that should have been visible immediately

## Example: motion

Stronger:
- transition explains a state change or reinforces feedback
- motion duration fits the product class

Weaker:
- surface feels busier but not clearer
- motion becomes the main personality because structure is weak
