# Hard Anti-Patterns

Use this file for patterns that should be treated as forbidden by default inside portable Frontend Taste guidance.

## Generic equal three-column card rows

Hard no by default.

Why:
- they flatten hierarchy
- they are one of the strongest generic-ai layout tells
- they encourage repetition without pacing
- they usually signal convenience instead of authored composition

A repo may still justify a narrow exception, but the burden of proof is high.

## Custom cursors

Hard no by default.

Why:
- they frequently make the surface feel gimmicky instead of refined
- they often draw attention to the cursor itself instead of to the interaction design
- they are rarely the real answer to a weak interaction concept

Exception rule:
- only allow when the target repo's design memory explicitly calls for it
- do not introduce it from taste preference alone
