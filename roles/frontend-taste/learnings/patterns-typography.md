# Typography Patterns

Use this file when the question is about headline strategy, hierarchy, text rhythm, expressive typography, or whether the type system is carrying the surface with enough control.

## Good typography jobs

Good typography should:
- establish hierarchy without needing constant decorative help
- make the product feel deliberate at first read
- control pace, emphasis, and density across the surface
- match the product class instead of defaulting to generic sameness or novelty

## Hierarchy patterns

### Controlled display hierarchy

Use when the surface needs a strong first read without becoming shouty.

Good signs:
- the first heading is strong but not doing all the work alone
- secondary headings, body, labels, and metadata all contribute to structure
- contrast comes from scale, weight, spacing, and tone together

Bad signs:
- one giant heading carries the whole screen
- everything else collapses into weak supporting text

### Editorial contrast

Use when the surface wants more atmosphere, cadence, or taste-led emphasis.

Good signs:
- contrast in type families or styles creates a stronger reading rhythm
- expressive type still preserves readability and structure
- serif or distinctive display treatment has a real compositional reason

Use carefully:
- editorial contrast can become costume quickly
- if the typography draws attention to itself more than to the content hierarchy, it is probably overdone

### Dense utility typography

Use when operational clarity matters more than theatrical expression.

Good signs:
- labels, values, table text, and controls stay highly scannable
- contrast is cleaner and tighter
- the system feels precise without becoming visually dead

Common fits:
- dashboards
- admin panels
- docs

## Typography do / don't

Do:
- build hierarchy through the full text system, not just the hero line
- choose type behavior that matches the product class
- use expressive display treatment selectively and intentionally
- keep readability intact when adding contrast or atmosphere

Don't:
- use oversized hero type as a substitute for hierarchy
- use decorative text effects to fake distinctiveness
- drop serif into dense software surfaces without a strong reason
- let kinetic or stylized typography overpower navigation, content, or action clarity

## Smells

Bad signs:
- giant H1 doing all the emotional work
- type effects replacing real structure
- decorative display choices that clash with the product class
- expressive type that looks strong in a screenshot but weakens sustained reading
- a typography system that feels either default-bland or insecurely over-styled
