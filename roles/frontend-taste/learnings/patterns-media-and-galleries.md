# Media and Gallery Patterns

Use this file when the question is about how imagery, video, galleries, carousels, or other media blocks should participate in hierarchy, pacing, and perceived quality.

## Good media jobs

Good media should:
- carry a clear compositional job rather than fill empty space
- strengthen hierarchy, atmosphere, or proof
- help pacing by changing the reading rhythm at the right moments
- feel selected and framed, not dumped into slots
- stay legible inside the surrounding layout, not fight it

If media does not add proof, atmosphere, or pacing, it is probably filler.

## Media lead vs media support

### Media-led sections

Use when the image, video, or visual world is one of the main reasons the section exists.

Good signs:
- the media asset contains real payoff, proof, or atmosphere
- text is edited to support the media instead of competing with it
- framing and crop choices feel authored

Common fits:
- marketing hero sections
- portfolio/project showcases
- editorial storytelling blocks
- product reveal moments

### Media-supporting sections

Use when structure, explanation, or data still carries the primary read.

Good signs:
- media reinforces understanding without becoming the whole section
- text and layout stay in control
- screenshots, diagrams, or product shots are scaled to support scanning

Common fits:
- feature explanation sections
- docs and help surfaces
- dashboard/product detail explanations

## Pattern families

### Editorial image band

Use when the page needs a strong pacing change or visual reset.

Good signs:
- one image or tightly related set creates a clean break in rhythm
- the band feels intentional, not like leftover filler between sections
- the surrounding spacing respects the weight of the media

### Structured gallery grid

Use when multiple assets need comparison or a broad visual read.

Good signs:
- the grid has a clear hierarchy instead of identical-image wallpaper
- asset size differences feel meaningful
- cropping and spacing stay disciplined

### Focused carousel or slider

Use when sequence, comparison, or constrained space genuinely justifies it.

Good signs:
- the carousel solves a real layout or narrative problem
- the current item is easy to understand
- controls and progression stay calm and predictable

Use carefully:
- carousels are often used to hide clutter or indecision
- if showing all key content at once would be clearer, the carousel is probably wrong

### Staggered or asymmetric media composition

Use when the section wants a stronger authored feel.

Good signs:
- asymmetry improves rhythm, emphasis, or narrative motion
- the layout still feels controlled and readable
- the composition does not rely on chaos to feel creative

## Media do / don't

Do:
- choose media with a real job in the section
- make crop, scale, and framing decisions feel authored
- let media change pacing when the page needs a rhythm shift
- use galleries when multiple assets add real understanding or proof

Don't:
- use media as generic filler behind weak structure
- add a carousel just to avoid choosing what matters most
- let every asset have equal visual weight by default
- use decorative distortion or effects to make ordinary assets seem more special

## Utility-surface note

For docs, dashboards, and admin surfaces:
- screenshots, diagrams, and product views should bias toward legibility first
- media usually supports explanation rather than dominating composition
- gallery logic should not slow scanning or hide obvious detail

## Smells

Bad signs:
- screenshot dumps with no hierarchy
- gallery layouts that feel like wallpaper instead of composition
- media blocks that look generated into slots rather than chosen
- over-cropped assets that lose the thing the user needed to see
- carousels hiding simple content that should have been visible immediately
- image-heavy sections with no stronger narrative or structural payoff
