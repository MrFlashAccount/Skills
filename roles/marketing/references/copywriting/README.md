# Copywriting Reference

Use for net-new market-facing copy or rewrites that materially change the message.

## Use when

- landing pages, homepages, pricing pages, feature pages, comparison pages, and campaign pages
- paid ad or paid social variants
- nurture-email blocks, launch-email sections, and sales one-pager sections
- headline, subheadline, CTA, objection-handling, proof framing, and section sequencing

## Do not use when

- the work is developer-facing README or technical docs
- the existing asset only needs polish and the strategy is already basically right

## Task classes

- `section-fix`: localized rewrite of a hero, CTA block, proof block, or short surface
- `full-cycle`: a new page or any rewrite that changes angle, audience emphasis, proof hierarchy, section order, or offer framing

## Workflow

- `section-fix`: contract -> surface check -> rewrite -> one marketing review
- `full-cycle`: contract -> audience/channel fit check -> message hierarchy -> section plan -> draft -> critic attack -> revise -> review checkpoints -> final packaging

## Rules

- Name the audience, awareness stage, and primary conversion action before drafting.
- Load shared GTM context before making positioning or promise decisions.
- Match the channel; pages, ads, lifecycle email, and sales collateral need different density and CTAs.
- Keep one primary message and one primary action per surface unless branching is intentional.
- Prefer customer-language clarity over slogan cleverness.
- Never invent proof, testimonials, metrics, or competitive wins.
