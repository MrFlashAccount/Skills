# Sequence Design

Use this reference when the main problem is choosing the right lifecycle shape.

## Common lifecycle jobs

### Welcome / activation
- confirm the action they just took
- deliver the promised value fast
- point to the first meaningful step
- remove the first obvious blocker

### Onboarding / adoption
- reinforce the next milestone, not every feature
- use proof where activation anxiety is highest
- coordinate with product UX so email supports, not duplicates

### Nurture / education
- teach the problem and the buying lens gradually
- earn the direct offer with useful context and proof
- segment by role, use case, or intent when possible

### Re-engagement / win-back
- lead with changed value, not guilt
- acknowledge inactivity without sounding punitive
- give a clean exit when interest is gone

## Sequence red flags

Stop and fix the flow if you see:
- multiple emails with the same job
- no clear exit condition
- every email asking for the same conversion too early
- copy that assumes more product knowledge than the audience has
- campaign sends colliding with onboarding, sales, or support motions

## Useful experiment levers

- entry criteria and segmentation
- delay between touches
- proof placement
- CTA friction
- branch logic after key actions or non-actions
