# Outbound Patterns

Use this file as a compact outbound checklist, not as a template dump.

## High-signal triggers

Prefer triggers that naturally connect to the pain you solve:
- funding, hiring, or team expansion
- launch, pricing, or packaging changes
- product, stack, or process change
- recent content, hiring post, or public opinion that reveals a real priority
- visible friction in the prospect journey or site flow

If the trigger does not strengthen the problem hypothesis, cut it.

## Good first-touch moves

- observation tied to an implication
- role-specific pain stated in plain language
- one proof point or constraint that makes the claim believable
- one low-friction ask that can be answered from a phone

## Follow-up angle rotation

Use a different job for each touch:
- touch 1: core problem and relevance
- touch 2: new proof or sharper implication
- touch 3: objection or alternative path
- touch 4: respectful closeout / breakup

## Cold-email red flags

Reject the draft if it depends on:
- generic compliments
- product-first openings
- more than one CTA
- multiple links or heavy formatting
- jargon standing in for value
- personalization that could be pasted onto any company in the segment

## Testing ideas

When the user wants experimentation, vary one thing at a time:
- trigger type
- pain framing
- proof choice
- CTA friction
- sequence length or delay
