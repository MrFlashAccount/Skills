# Copy Editing Reference

Use for existing market-facing copy.

## Use when

- tightening or polishing page copy
- proof, objection, or CTA improvements
- outdated-page refreshes
- copy sweeps across landing pages, pricing pages, emails, sales one-pagers, or campaign assets

## Do not use when

- the page or campaign needs net-new strategy or a major rewrite
- the artifact is developer-facing README or technical docs

## Task classes

- `polish`: local edits with the same audience, angle, and structure
- `sweep`: multi-pass edit where the message mostly stays but clarity and proof need work
- `refresh`: published or older copy that may need factual, proof, or positioning updates

## Workflow

- `polish`: contract -> edit -> one review
- `sweep`: contract -> baseline read -> editing passes -> review checkpoint -> fix
- `refresh`: contract -> refresh-vs-rewrite decision -> editing passes -> factual accuracy check

## Rules

- Preserve the real strategic goal unless a new angle is explicitly requested.
- Load GTM context before judging audience fit or brand-language drift.
- Do not upgrade weak proof into strong claims.
- Flag unsupported metrics, testimonials, customer language, and competitive assertions.
- If a claim cannot be proved, soften it or mark the proof gap.
- Keep edits traceable so changes and reasons stay visible.
