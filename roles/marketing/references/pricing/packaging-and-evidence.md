# Packaging and evidence

Use this reference to keep pricing decisions grounded.

## 1. Separate the three pricing decisions

### Packaging
What is included in each offer?
- features
- usage limits
- service/support level
- access controls, security, implementation, or admin surfaces

### Value metric
What does the customer pay against?
- seat / user
- usage or consumption
- project, workspace, account, contact, record, or transaction
- flat subscription
- hybrid model

A strong metric scales when customer value scales and is easy to understand.

### Price architecture
How do amounts and upgrade paths work?
- number of tiers
- monthly vs annual structure
- overages or usage expansion
- enterprise/custom path
- discounts, grandfathering, pilot, or trial logic

## 2. Use evidence in an order that resists guesswork

Prefer this ladder:
1. direct customer language and interview evidence
2. observed usage and expansion patterns
3. sales and support friction patterns
4. win/loss notes and competitive comparisons
5. internal hypotheses labeled as hypotheses

If you only have level 5, say so and reduce confidence.

## 3. Package for fit, not for everyone

Good pricing usually makes fit boundaries clearer.

Name:
- who belongs in the entry tier
- who should move to the middle tier
- what justifies premium or enterprise treatment
- which customers are a poor fit for this structure

## 4. Pressure-test the message

For each tier or model, ask:
- what job is this for?
- what proof supports the price?
- what objection will appear first?
- what tradeoff are we asking the buyer to accept?
- what upgrade event should feel natural, not punitive?

## 5. When evidence is incomplete

Choose one of these instead of pretending certainty:
- propose a narrower beta or pilot package
- recommend customer-research work
- recommend sales-call observation or lost-deal review
- recommend a reversible rollout with explicit check metrics
