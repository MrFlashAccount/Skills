# Copywriting

Use this skill as the workflow harness for market-facing copy.

Start with the Marketing role and shared GTM context before drafting. This skill owns the copy workflow, page/surface structure checks, and output packaging.

## Routing boundary

Use this skill when the artifact is market-facing copy and the main job is persuasion, positioning-in-context, message hierarchy, or conversion.

Use it for:
- landing pages, homepages, pricing pages, feature pages, comparison pages, campaign pages
- paid ad copy, paid social copy, sponsored-post variants
- nurture-email blocks, launch-email sections, sales one-pagers, outbound snippets
- headline, subheadline, CTA, objection-handling, proof framing, section sequencing

Do not use it for:
- repository README openings, developer-facing feature framing, or product-facing technical entrypoints; use `devrel-copywriter`
- setup docs, tutorials, API docs, migration docs, or usage explanation; use `docs-writer`
- pure polish of existing market-facing copy when the strategy and structure are already basically right; use `copy-editing`

## Task class

- `section-fix`: localized rewrite of a hero, CTA cluster, proof block, objection block, or one short surface without changing the full strategy.
- `full-cycle`: a new page/surface or any rewrite that changes angle, audience emphasis, proof hierarchy, section order, offer framing, or channel adaptation.

If unsure, treat it as `full-cycle`.

## Workflow

- `section-fix`: contract -> surface check -> rewrite -> one marketing review -> final packaging.
- `full-cycle`: contract -> audience/channel fit check -> message hierarchy -> section plan -> draft -> marketing critic attack -> revise -> review checkpoint 1 -> fix -> review checkpoint 2 -> final packaging.

## Rules

- Treat `../../roles/marketing/ROLE.md` as the canonical judgment for audience fit, proof, differentiation, and claim discipline.
- Load shared GTM context before making positioning or promise decisions.
- Keep market-facing work separate from DevRel/docs work even when the same product is involved.
- Name the audience, awareness stage, and primary conversion action before drafting.
- Do not invent proof, customer language, testimonials, metrics, or competitive wins.
- If proof is weak, soften the claim or leave an explicit proof gap.
- Match the channel: ad copy, landing pages, lifecycle email, and sales collateral do not use the same density or CTA style.
- Keep one primary message and one primary action per surface unless the artifact explicitly needs branching.
- Prefer customer-language clarity over slogan cleverness.
