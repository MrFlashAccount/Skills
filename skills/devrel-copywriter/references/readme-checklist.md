# Repository readme checklist

Use this only for repository `readme` work.

This is a surface-level final check. The canonical quality doctrine lives in `../../roles/dev-rel/ROLE.md`, `../../roles/dev-rel/RUBRIC.md`, and `../../roles/dev-rel/LEARNINGS.md`.

## Final check

### First screen
- immediately clear what this is
- immediately clear who it is for
- immediately clear what problem it solves
- includes a credible pitch, not generic filler

### Product understanding
- explains what the product does
- explains why it exists or why it matters
- does not require reading half the file to understand the point

### First-value path
- shows how to try, connect, or use it
- quickstart or first-value steps were checked against the repo, source, config, examples, or docs
- expected outcome is clear enough to trust

### Structure and navigation
- chosen structure matches the audience and product
- examples or deeper links appear where they remove ambiguity fast
- deeper docs are linked instead of stuffed into the opening

### Content discipline
- avoids implementation-status dumps in the opening
- keeps caveats and deep internals lower unless immediately blocking
- does not pretend to replace deep documentation
