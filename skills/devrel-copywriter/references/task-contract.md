# Devrel copy contract

Build a short contract before drafting.

## Required fields

- `Artifact`: repository `readme`, README opening, or repo-entrypoint rewrite.
- `Audience`: who this is for and how technical they are.
- `Surface`: where it will live.
- `Goal`: what the reader should understand, feel, or do.
- `Scope`: what is in and out.
- `Structure`: the intended shape.
- `Proof points`: concrete facts, examples, constraints, screenshots, code reality.
- `Allowed claims`: claims supported by evidence.
- `Risky claims`: things that sound attractive but are not well-supported.
- `Tone`: desired voice and anti-voice.

## Output shape

Use this compact template when useful:

- Artifact
- Goal
- Audience
- Surface
- Scope / Non-goals
- Structure
- Proof points
- Risks / Unknowns
- Tone

## Critique prompts

Use these before drafting:
- What is the real payoff in one sentence?
- What would a skeptical developer not believe here?
- Where is the copy trying to sound smart instead of useful?
- What line would feel embarrassing if the product underdelivers?
- What should be shown later, not in the opening?
