# Documentation modes

Use this file to choose or confirm the primary doc mode before you lock the contract. Even tiny edits should name the current or target mode and verify the patch stays inside it. Do not blur modes by accident.

## Quick start

- Optimize for the fastest credible first win.
- Keep choices to a minimum.
- Avoid deep architecture.
- One narrow example beats a broad tour.

## Tutorial / getting started

- Lead the reader through one concrete outcome.
- Explain just enough context to keep the steps meaningful.
- Keep the order strict.
- Avoid branching unless necessary.

## How-to guide

- Assume the reader knows the basics.
- Solve one practical task.
- Focus on the decisions and caveats that matter for that task.

## Reference

- Be complete, structured, and easy to scan.
- Prioritize exactness over narrative flow.
- Keep examples short and local.

## Explanation

- Teach the mental model.
- Clarify why things are shaped this way.
- Keep this separate from the first-run path unless the model is required immediately.
