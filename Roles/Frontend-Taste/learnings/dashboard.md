# Dashboard

Use for app-like analytical surfaces where scan speed and density matter.

## Defaults

- higher information density
- tighter but still legible spacing
- strong table/card/filter hierarchy
- clarity of state and interaction over visual theater
- sans-first typography unless the repo explicitly calls for something else

## What good looks like

- fast scanability
- clear grouping of controls, data, and actions
- density tuned for working use, not gallery aesthetics
- interactions feel obvious before they feel beautiful
- containers used deliberately instead of by habit

## Common failures

- wasting space that forces unnecessary scrolling
- decorative chrome that competes with data
- weak distinction between primary and secondary controls
- marketing-page spacing on utility surfaces
- ornamental motion or stylization that slows operational reading

## Common useful pattern families

- divider-led grouping
- dense utility stacks
- selective cards around modules that genuinely need containment
- state treatments that preserve layout structure while data changes
