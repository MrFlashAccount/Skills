# Pattern Families: States

Use this file when the question is about loading, empty, error, success, or transition quality as part of presentation quality.

## Loading states

Good loading states:
- preserve the expected structure of the final surface
- communicate where information will appear
- feel calm and intentional rather than noisy or generic

Bad loading states:
- generic spinner with no structural clue
- loading treatment that changes layout dramatically when content arrives

## Empty states

Good empty states:
- explain what is missing
- suggest the next useful action
- keep the surrounding composition coherent

Bad empty states:
- blank voids
- cute illustrations with no guidance
- oversized empty-state theater in a utility surface

## Error states

Good error states:
- are visible where the problem matters
- do not collapse the rest of the page into chaos
- clearly separate recoverable issues from hard failures

Bad error states:
- tiny red text lost in the layout
- giant destructive tone for routine validation errors

## Success and confirmation feedback

Good confirmation feedback:
- feels responsive and reassuring
- stays proportionate to the action
- does not demand more attention than the completed task deserves

Bad confirmation feedback:
- celebratory noise for ordinary actions
- weak feedback that leaves the user unsure whether anything happened
